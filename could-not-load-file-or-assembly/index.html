Could not load file or assembly.... OfficePnP Nuget package conflict issues

By admin

June 27, 2019

If you are using OfficePnP or SharePoint PnP nuget packages in your solution, often you end up with DLL conflicts and may throw error “Could not load file or assembly…” I have faced this many times and was struggling to fix this. This article is about how to solve the dll conflicts while adding OfficePnP nuget to your solution.

What is the problem?

Most of the cases, we may want to utilize the OfficePnP solution customize our SharePoint sites. Recently I was working on project where I had to create modern sharepoint team sites and then add modern webparts like Welcome text, Highlighted Content , Quick links with static content etc as part of provisioning (if anyone interested in this solution, please leave a comment). In this case, I have built a custom solution and then added OfficePnP nuget to manage the modern pages.

When I built the project, I was getting lot of issues with error message could not load file or assembly…â€¦ . This was happening for almost all the Dlls which are common in OfficePnP and my solution

How can we solve this ?

The simplest solution for this is to add the nuget to the custom project where OfficePnP is referred. For eg:- When you run your project say it throws error “Could not load file or assembly System.Net.http 4.1.1…..” , stop the execution and add the nuget package System.Net.Http 4.1.1 or latest to your project. Build again and execute and you can see that the error disappeared. You may have to repeat this for all the other dlls which throws this error.
